# Flora Sanctum 开发决策记录

> 记录在开发过程中遇到的问题、推测和决策。供后续 Agent 与评审追溯。

## 目标

按 spec 完成 `flora-sanctum` 密码管理工具的全部阶段：脚手架 → 加密核心 → 数据模型与存储 → JGit 同步 → GUI → 打包。

## 决策格式

| 日期 | 决策/问题 | 可选方案 | 选择与理由 | 影响 |
|------|----------|----------|-----------|------|
| 2026-07-23 | JavaFX 版本选择 | 26.0.2 / 26.0.1 / 23.0.2 | 26.0.2 在阿里云镜像中不可用（缺少 Linux 原生 jar），回退到 26.0.1。该版本有完整的 Linux jar，且与 JDK 26 兼容。 | pom.xml 中 javafx.version = 26.0.1 |
| 2026-07-23 | JGit 引入策略（修订） | Maven 依赖 / 源码 vendor（966 文件）/ git CLI ProcessBuilder | **放弃 vendor JGit，改用 JDK ProcessBuilder 调用系统 git CLI。** 理由：(1) JGit 核心模块 966 源文件 + 3 个传递依赖包（slf4j、javaewah、commons-codec）需同步 vendor；(2) JPMS 模块体系下与无 module-info 的旧代码兼容困难；(3) git CLI 已部署在 Linux/macOS/Windows，ProcessBuilder 是 JDK 内置能力，零依赖且简单可靠。Flora Sanctum 所需的 4 个操作（init、add/commit、push、pull）用 CLI 实现更可维护。 | 删除已复制的 JGit 源码；同步层改用 GitBackend（ProcessBuilder 封装）；更新 module-info.java 移除 JGit 相关注释；在 THIRD-PARTY-NOTICES 中移除 JGit 条目 |
| 2026-07-23 | Git init 目录不存在问题 | gitExec 直接调用 / 先创建目录再调用 | `git -C <dir>` 要求目标目录已存在。改为 `gitExec(workingDir, "git", "init")` 并先创建目录。 | GitBackend.init() 修改 |
