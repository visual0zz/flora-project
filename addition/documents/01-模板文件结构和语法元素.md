# 01 — 模板文件结构和语法元素

## 文件结构

一个 `.ramet` 文件包含两个部分：

```
<#meta>                        ← Meta 标签块
@Param{ ... }
@Cartesian{ ... }
@Path{ ... }
@SkipWhen{ ... }
</#meta>
模板正文...                    ← 渲染内容
```

元数据块必须位于模板第一行的 `<#meta>...</#meta>` 块中，声明渲染参数、笛卡尔积轴、输出路径和跳过条件。

## 两区域划分

模板正文分为两种区域：

### 被动区域

所有不在 `<#...>` 和 `${}` 内的文本。

- **零转义**，所有字符原样输出
- `\`、`$`、`<`、`#` 无特殊含义
- 如需在输出中包含 `${` 或 `<#`，使用 `${"${"}` 或 `${"<#"}`

### 逻辑区域

`<#...>` 指令内部和 `${}` 插值内部。

- 无语法级转义
- 字符串字面量 `"..."` 内部支持 Java 风格转义

## 表达式语法

表达式使用 Lson 语法，类似 JSON + 函数调用。

| 写法 | 说明 |
|------|------|
| `42` / `3.14` | 数字 |
| `"hello"` | 字符串（支持 Java 转义） |
| `true` / `false` / `null` | 关键字 |
| `[1, 2, 3]` | 数组 |
| `{ k: "v" }` | 对象（键不能是 true/false/null） |
| `a.b` | 属性链 |
| `arr[0]` | 索引访问 |
| `f(x, y)` | 函数调用 |
| `a gt b` | 中缀函数 |
| `not a` | 前缀函数 |

### 中缀/前缀说明

内置函数名可作为中缀或前缀运算符使用：

```ramet
a greaterThan b          → greaterThan(a, b)
a and b                  → and(a, b)
not a                    → not(a)
```

**中缀无优先级**：`a greaterThan b and c greaterThan d` 解析为 `greaterThan(and(greaterThan(a,b),c),d)`。需用括号控制优先级：`(a greaterThan b) and (c greaterThan d)`。

### 字符串转义

在 `"..."` 内部支持以下转义：

| 转义 | 含义 |
|------|------|
| `\"` | 双引号 |
| `\\` | 反斜杠 |
| `\n` | 换行 |
| `\r` | 回车 |
| `\t` | 制表符 |
| `\b` | 退格 |
| `\f` | 换页 |
| `\'` | 单引号 |
| `\uXXXX` | Unicode（4 位十六进制） |

### 引用与求值

- 未加引号的标识符（如 `name`）视为变量引用，从渲染上下文中查找
- 属性链 `a.b.c` 访问嵌套属性
- 索引 `arr[0]` 访问列表元素
